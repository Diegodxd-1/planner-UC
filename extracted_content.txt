“Año de la Esperanza y el Fortalecimiento de la Democracia” Taller de Proyectos 2 - Ingeniería de Sistemas e Informatica Docente:  Job Daniel Gamarra Moreno Integrantes: ARAUJO HUAMANI, Leonardo Daniel CURI UNTIVEROS, Jefferson Diego VASQUEZ MIRANDA, Luis Alexis VILCARANO DE LA CRUZ, Frank Anthony NRC:  28574 Huancayo - Perú 2026 Proyecto de Fin de Asignatura (PFA) DOCUMENTO INICIAL DEL PROYECTO (PRIMER BORRADOR) Contexto del problema En las universidades, la planificación de horarios académicos es una tarea compleja que involucra múltiples variables como la disponibilidad de docentes, la asignación de aulas, los cursos ofrecidos, las secciones disponibles y las restricciones académicas asociadas. Generalmente, este proceso se realiza de forma manual o mediante herramientas poco automatizadas, lo que puede generar errores, conflictos de horarios y dificultades en la organización académica. Además, la coordinación de estos elementos requiere considerar diversas restricciones, como la capacidad de las aulas, la disponibilidad horaria de los docentes, los prerrequisitos entre cursos y la distribución adecuada de las cargas académicas. La ausencia de herramientas adecuadas para gestionar estos factores puede provocar solapamientos de horarios, uso ineficiente de recursos y retrasos en la planificación académica. Problema central El problema central consiste en la dificultad de generar horarios académicos universitarios de manera eficiente, considerando múltiples restricciones y recursos limitados. La generación manual de horarios suele ser propensa a errores, conflictos entre cursos, docentes o aulas, y puede requerir múltiples ajustes antes de obtener una solución válida. Esta situación genera pérdida de tiempo en los coordinadores académicos y puede afectar la correcta planificación de las actividades educativas. Ambigüedad del problema No se especifica el número exacto de bloques horarios disponibles en el sistema. No se define la prioridad entre las diferentes restricciones cuando ocurre un conflicto. No se detalla si los docentes pueden dictar múltiples secciones del mismo curso en diferentes horarios. No se establece si los estudiantes pueden elegir libremente sus cursos o si siguen un plan de estudios predefinido. No se define el número máximo de cursos que un estudiante puede llevar por periodo académico. Variables involucradas El problema de generación de horarios involucra diferentes variables que deben ser consideradas para construir una solución válida. Entre las principales variables se encuentran: Estudiantes matriculados en los cursos. Docentes responsables de cada asignatura. Cursos y secciones disponibles en el periodo académico. Aulas disponibles y su capacidad. Bloques horarios definidos para el funcionamiento académico. Restricciones asociadas a disponibilidad y compatibilidad. Estas variables deben combinarse adecuadamente para evitar conflictos y garantizar una planificación académica funcional. Restricciones del problema El proceso de generación de horarios debe respetar diversas restricciones, entre ellas: Un docente no puede estar asignado a dos cursos en el mismo horario. Un aula no puede ser utilizada por más de un curso simultáneamente. Los estudiantes no deben tener cursos obligatorios en el mismo horario. La capacidad del aula debe ser suficiente para el número de estudiantes. Deben respetarse los prerrequisitos entre cursos. Estas restricciones hacen que la generación de horarios sea un problema complejo que requiere un enfoque sistemático para su resolución. Necesidad de una solución tecnológica Debido a la complejidad del problema, resulta necesario desarrollar una herramienta tecnológica que permita automatizar el proceso de generación de horarios académicos. Una aplicación web moderna puede facilitar la gestión de datos académicos, la validación de restricciones y la generación automática de soluciones válidas. El desarrollo de esta herramienta permitirá mejorar la eficiencia en la planificación académica, reducir errores humanos y optimizar el uso de recursos institucionales. Alcance del proyecto Como parte del proyecto del curso, se desarrollará un  Producto Mínimo Viable (PMV)  de una aplicación web para la generación de horarios universitarios. Este sistema permitirá registrar la información básica del problema, aplicar restricciones académicas y generar horarios de forma automática. El sistema será desarrollado utilizando el  stack tecnológico MERN (MongoDB, Express, React y Node.js)  y se enfocará en demostrar la viabilidad de una solución tecnológica para el problema planteado.
