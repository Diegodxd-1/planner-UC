# Project Charter

**Taller de Proyectos 2 - Ingeniería de Sistemas e Informática**

---

## 1. Título del Proyecto
**SISTEMA DE GENERACIÓN ÓPTIMA DE HORARIOS ACADÉMICOS EN ENTORNOS DE CURRÍCULO FLEXIBLE**

## 2. Carta del Proyecto
El presente proyecto tiene como finalidad desarrollar una solución que permita generar horarios académicos óptimos en entornos universitarios con currículo flexible. Este tipo de entorno presenta alta complejidad debido a la variabilidad en la matrícula, restricciones académicas y disponibilidad de recursos como docentes y aulas.

El sistema propuesto busca automatizar el proceso de asignación de horarios considerando múltiples restricciones y condiciones, permitiendo obtener soluciones válidas que eviten conflictos y optimicen la distribución de recursos.

## 3. Detalles del Proyecto

### Descripción del producto / Entregables
Desarrollo de un sistema que permita generar horarios académicos de forma automática, considerando múltiples restricciones como disponibilidad de docentes, aulas, cursos y estudiantes.

**Entregables:**
* Documento de análisis del problema
* Modelo del problema (restricciones y variables)
* Diseño de la solución
* Sistema funcional de generación de horarios
* Pruebas del sistema
* Informe final del proyecto
* Video demostrativo

### El proyecto NO incluye
* Implementación en sistemas reales de universidades
* Integración con sistemas institucionales existentes
* Infraestructura adicional
* Capacitación formal a usuarios finales
* Soporte posterior a la entrega

### Recursos pre-asignados
* Equipo de estudiantes de ingeniería de sistemas
* Computadoras personales
* Acceso a herramientas de desarrollo
* Tiempo asignado para desarrollo del proyecto
* Conocimiento básico del dominio académico

## 6. Consideraciones del proyecto

### Riesgos de alto nivel
* Complejidad del problema debido a múltiples restricciones.
* Cambios en los requerimientos durante el desarrollo.
* Limitaciones en tiempo y recursos.
* Dificultad para obtener soluciones óptimas.
* Falta de experiencia en modelado del problema.

### Criterios de aceptación
* El sistema genera horarios sin conflictos.
* Se cumplen las restricciones principales.
* Los horarios son válidos y utilizables.
* El sistema funciona correctamente en pruebas.
* Se entrega toda la documentación requerida.

### Suposiciones
* Se contará con información suficiente para modelar el problema.
* Los datos de entrada serán válidos.
* El equipo tendrá disponibilidad para desarrollar el proyecto.
* Las restricciones se mantendrán estables.

### Restricciones
* Tiempo limitado para el desarrollo.
* Recursos técnicos limitados.
* Proyecto desarrollado solo por estudiantes.
* No se contará con datos reales completos.
* Alcance limitado a un prototipo funcional.
