# Formulario aplicado - System Usability Scale (SUS)

## Contexto

Sistema evaluado: Planner-UC.

Fecha de aplicacion: 2026-06-15.

Modalidad: prueba controlada guiada, con ejecucion de tareas y cuestionario posterior.

Escala de respuesta:

1. Totalmente en desacuerdo
2. En desacuerdo
3. Neutral
4. De acuerdo
5. Totalmente de acuerdo

## Tareas previas al cuestionario

1. Iniciar sesion con una cuenta asignada.
2. Revisar el dashboard y ubicar los modulos disponibles.
3. Crear o editar un curso.
4. Crear o editar un aula.
5. Revisar la lista de usuarios y ubicar la accion de edicion.
6. Abrir el generador de horarios e interpretar el estado del backend.
7. Cerrar sesion.

## Preguntas SUS aplicadas

| Codigo | Enunciado |
| --- | --- |
| Q1 | Creo que me gustaria usar este sistema con frecuencia. |
| Q2 | Encontre el sistema innecesariamente complejo. |
| Q3 | Pense que el sistema era facil de usar. |
| Q4 | Creo que necesitaria ayuda tecnica para poder usar este sistema. |
| Q5 | Encontre que las funciones del sistema estaban bien integradas. |
| Q6 | Pense que habia demasiada inconsistencia en el sistema. |
| Q7 | Imagino que la mayoria de personas aprenderian a usar este sistema rapidamente. |
| Q8 | Encontre el sistema dificil de usar. |
| Q9 | Me senti seguro usando el sistema. |
| Q10 | Necesite aprender muchas cosas antes de poder usar el sistema. |

## Criterio de calculo

Para preguntas impares se calcula `respuesta - 1`.

Para preguntas pares se calcula `5 - respuesta`.

El puntaje SUS individual es:

`sumatoria de contribuciones * 2.5`

El puntaje final del sistema corresponde al promedio de los puntajes individuales.
